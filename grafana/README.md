# Rock 88.7 — Grafana Observability Stack

This directory holds the Grafana **provisioning** (`provisioning/`) and **dashboard JSON** (`dashboards/`) used by the `docker-compose.observability.yml` stack at the repository root. Bringing the whole observability stack up is a single command — run `docker compose -f docker-compose.observability.yml --profile observability up -d` from the project root; that boots **Prometheus** (scraping `http://host.docker.internal:3000/api/v1/metrics` every 10 s — edit `prometheus.yml` to use `localhost:3000` if you run the Next.js app natively), **Grafana** (auto-provisioning the Prometheus datasource + the four Rock 88.7 dashboards on first boot, exposed on http://localhost:3001, log in with `admin` / `admin`), and **Alertmanager** (port 9093, config read from `./alertmanager/alertmanager.yml`). The dashboards — *Broadcast Overview*, *Audio Quality & Compliance*, *AI Module Cost & Performance*, and *Incidents & Reliability* — appear automatically under the **Rock 88.7** folder in Grafana; the first one is set as the home dashboard via `GF_DASHBOARDS_DEFAULT_HOME_DASHBOARD_PATH`.

> Panels that reference metrics not yet emitted by `src/app/api/v1/metrics/route.ts` (e.g. `listeners_total`, `audio_lufs_integrated`, `ai_module_runs_total`, `incidents_total`, `audit_log_events_total`) are written against the same naming convention as the currently-exposed `events_total` / `webhook_deliveries` / `uptime_seconds` family — they will populate automatically as the corresponding instrumentation is added to `lib/event-bus.ts` / route handlers, no dashboard edits required.
