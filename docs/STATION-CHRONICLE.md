## How This File Should Be Maintained

1. **Never delete entries** — even superseded ones. This is a chronicle, not a database.
2. **Date every entry** — precision matters (which month, which quarter).
3. **Be honest** — if AI failed, record it. If a decision was wrong, say so.
4. **Include numbers** — "ALT improved" is weak. "ALT went from 18.9 to 22.3" is strong.
5. **Include human reactions** — "Operator said AI is usually right" is more valuable than any metric.
6. **Write in the station's language** — this is for the station, not for GitHub.
7. **Review annually** — at the end of each year, read the whole chronicle and reflect.
